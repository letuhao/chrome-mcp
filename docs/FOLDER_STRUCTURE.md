## ğŸ“‚ Folder Descriptions

### `/src`
Source code for the MCP server. Contains TypeScript files that are compiled to JavaScript.

### `/dist`
Compiled JavaScript output (generated by `npm run build`). This folder is gitignored as it's generated from source.

### `/docs`
All project documentation. See [docs/README.md](README.md) for documentation index.

### `/scripts`
- **Root**: Core scripts used by the MCP server
  - `download-newest-torrent.js`: Basic torrent selection logic
  - `download-newest-torrent-enhanced.js`: Enhanced version with outdated torrent support
  
- **helpers/**: Utility scripts for testing and debugging
  - `test-mcp-connection.js`: Test MCP server and Chrome connection

- **startup/**: Scripts to start Chrome with remote debugging
  - Windows batch and PowerShell scripts for different scenarios

### `/logs`
Runtime logs and reports generated by automation:
- Daily logs: `auto-download-YYYY-MM-DD.log`
- Batch reports: `batch-report-*-*.json`
- Statistics: `stats.json`

### `/reports`
Temporary verification and test reports (gitignored).

### `/node_modules`
NPM dependencies (gitignored).

## ğŸ”’ Git Ignored Files

The following are excluded from version control:
- `node_modules/` - Dependencies
- `dist/` - Compiled output
- `logs/` - Runtime logs
- `reports/` - Test/verification reports
- `*.log` - Log files
- `package-lock.json` - Lock file (optional, depends on project preference)
- Temporary files, IDE configs, etc.

See [.gitignore](../.gitignore) for complete list.

## ğŸ“ File Naming Conventions

- **Source files**: `kebab-case.ts` (e.g., `download-newest-torrent.js`)
- **Documentation**: `UPPER_CASE.md` or `kebab-case.md`
- **Scripts**: `kebab-case.js` or `kebab-case.ps1`
- **Logs**: `feature-YYYY-MM-DD.log` or `feature-*.log`
- **Reports**: `feature-report-YYYY-MM-DD.json` or `*-report*.json`
